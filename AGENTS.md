Проект на FSD + TypeScript. Разделяй бизнес‑логику и UI‑логику: не помещай логику в UI‑компоненты и не смешивай UI в логические/функциональные блоки. Стремись к максимальному упрощению.

Для логики используй кастомные хуки и придерживайся принципов чистого кода.

Всегда используй `StateProvider` для обработки `isLoading` / `isError` / `empty` вместо ручных проверок и прямого рендера `LoadingState` / `ErrorState`.

Комментарии — минимальные, только там, где без них теряется смысл для других разработчиков.
