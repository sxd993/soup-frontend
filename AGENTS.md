Проект на FSD + TypeScript. Разделяй бизнес‑логику и UI‑логику: не помещай логику в UI‑компоненты и не смешивай UI в логические/функциональные блоки. Стремись к максимальному упрощению.

Для логики используй кастомные хуки и придерживайся принципов чистого кода.

Всегда используй `StateProvider` для обработки `isLoading` / `isError` / `empty` вместо ручных проверок и прямого рендера `LoadingState` / `ErrorState`. Скелетоны также задаются через `StateProvider` (проп `loadingComponent`).

Комментарии — минимальные, только там, где без них теряется смысл для других разработчиков. Существующие комментарии в коде удалять нельзя.

Если я предложил какую то задачу и описал ее — ты перед тем как ее начать — проанализируй ее и если есть вариант лучше, то предлагай, обсудим перед началом работы.

Если компонент находится в ui папке, то ты обязан выносить все хелперы, функции, чтобы в ui компоненте оставалось только return обычный html, вся логика в компонент должна прилетать через хуки, но не через пропсы! Нужно чтобы компонент внутри использовать просто хук. Избегай проп-дриллинга
